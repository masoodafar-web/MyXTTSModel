🚀 شروع سریع (Quick Start)

نصب وابستگی‌ها (Installation)
قبل از شروع آموزش، باید تمام کتابخانه‌های مورد نیاز را نصب کنید.
# نصب کتابخانه‌های پایه pip install -r requirements.txt # نصب پکیج MyXTTS pip install -e .
✅ توجه: اطمینان حاصل کنید Python 3.8 یا بالاتر نصب شده باشد.
اولین آموزش (First Training)
ساده‌ترین روش برای شروع آموزش مدل.
# آموزش پایه با تنظیمات پیش‌فرض python3 train_main.py \ --train-data ../dataset/dataset_train \ --val-data ../dataset/dataset_eval
💡 نکته: این دستور با تنظیمات پیش‌فرض شروع به آموزش می‌کند. برای کنترل بیشتر از پارامترهای اضافی استفاده کنید.
بررسی پیشرفت (Progress Monitoring)
در حین آموزش، می‌توانید پیشرفت را در TensorBoard مشاهده کنید.
# اجرای TensorBoard tensorboard --logdir runs/ # دسترسی از طریق مرورگر # http://localhost:6006

📚 آموزش پایه (Basic Training)

ساختار دیتاست (Dataset Structure)
دیتاست باید ساختار خاصی داشته باشد تا مدل بتواند آن را بخواند.
dataset/ ├── dataset_train/ │ ├── wavs/ # فایل‌های صوتی │ │ ├── speaker1_001.wav │ │ ├── speaker1_002.wav │ │ └── ... │ └── metadata.txt # فایل متنی حاوی متن‌ها │ └── dataset_eval/ ├── wavs/ └── metadata.txt
فرمت metadata.txt:
speaker1_001|سلام دنیا|Hello world
speaker1_002|این یک مثال است|This is an example
پارامترهای اصلی آموزش
جدول زیر مهم‌ترین پارامترهای آموزش را نشان می‌دهد.
پارامتر توضیحات مقدار پیش‌فرض
--train-data مسیر دیتاست آموزش ضروری
--val-data مسیر دیتاست اعتبارسنجی ضروری
--batch-size تعداد نمونه در هر batch 8
--epochs تعداد epoch های آموزش 100
--lr نرخ یادگیری (Learning Rate) 0.0001
--checkpoint-dir مسیر ذخیره checkpoint ها checkpointsmain/
مثال آموزش کامل
python3 train_main.py \ --train-data ../dataset/dataset_train \ --val-data ../dataset/dataset_eval \ --batch-size 16 \ --epochs 200 \ --lr 0.0002 \ --checkpoint-dir ./my_checkpoints \ --save-interval 10

⚡ آموزش پیشرفته (Advanced Training)

بهینه‌سازی GPU (GPU Optimization)
برای استفاده بهینه از GPU و افزایش سرعت آموزش از این تنظیمات استفاده کنید.
python3 train_main.py \ --train-data ../dataset/dataset_train \ --val-data ../dataset/dataset_eval \ --optimization-level enhanced \ --enable-gpu-stabilizer \ --batch-size 32 \ --gradient-accumulation-steps 4
💡 نکته:
  • --optimization-level enhanced: فعال‌سازی بهینه‌سازی‌های پیشرفته
  • --enable-gpu-stabilizer: پایدارسازی استفاده از GPU
  • --gradient-accumulation-steps: شبیه‌سازی batch بزرگ‌تر با حافظه کمتر
آموزش با Mixed Precision
استفاده از mixed precision برای سرعت بیشتر و مصرف حافظه کمتر.
python3 train_main.py \ --train-data ../dataset/dataset_train \ --val-data ../dataset/dataset_eval \ --mixed-precision \ --optimization-level enhanced
✅ مزایا:
  • سرعت آموزش 2-3 برابر بیشتر
  • مصرف حافظه 30-50% کمتر
  • امکان استفاده از batch size بزرگ‌تر
آموزش از checkpoint قبلی
ادامه آموزش از یک checkpoint ذخیره شده.
python3 train_main.py \ --train-data ../dataset/dataset_train \ --val-data ../dataset/dataset_eval \ --checkpoint checkpointsmain/checkpoint_epoch_50.pt \ --resume-training
تنظیمات پیشرفته Loss
کنترل دقیق وزن‌های مختلف loss برای بهبود کیفیت.
python3 train_main.py \ --train-data ../dataset/dataset_train \ --val-data ../dataset/dataset_eval \ --mel-loss-weight 1.0 \ --duration-loss-weight 0.5 \ --speaker-loss-weight 0.3 \ --stop-loss-weight 0.2
💡 راهنمای تنظیم Loss Weights:
  • mel-loss-weight: وزن اصلی (همیشه 1.0 نگه دارید)
  • duration-loss-weight: برای سرعت بیان مناسب (0.3-0.7)
  • speaker-loss-weight: برای شبیه‌سازی صدا (0.2-0.5)
  • stop-loss-weight: برای پایان صحیح جمله (0.1-0.3)

🔧 بهینه‌سازی (Optimization)

انتخاب Batch Size مناسب
انتخاب batch size مناسب بر اساس حافظه GPU شما.
حافظه GPU Batch Size توصیه شده توضیحات
4-6 GB 4-8 GPU های ضعیف‌تر
8-12 GB 8-16 GPU های متوسط (RTX 2060, 3060)
16-24 GB 16-32 GPU های قوی (RTX 3080, 3090, 4090)
24+ GB 32-64 GPU های حرفه‌ای (A100, V100)
استراتژی Learning Rate
انتخاب استراتژی مناسب learning rate برای همگرایی بهتر.
# استفاده از Learning Rate Scheduler python3 train_main.py \ --train-data ../dataset/dataset_train \ --val-data ../dataset/dataset_eval \ --lr 0.0005 \ --lr-scheduler cosine \ --warmup-steps 1000
💡 انواع Scheduler:
  • cosine: کاهش نرم و تدریجی (توصیه می‌شود)
  • plateau: کاهش زمانی که loss ثابت شد
  • step: کاهش در epoch های مشخص
  • exponential: کاهش نمایی
Gradient Clipping
جلوگیری از exploding gradients با محدود کردن مقدار گرادیان.
python3 train_main.py \ --train-data ../dataset/dataset_train \ --val-data ../dataset/dataset_eval \ --gradient-clip-val 1.0 \ --gradient-clip-norm 2.0
⚠️ هشدار: اگر loss به NaN تبدیل می‌شود، مقدار gradient clipping را کاهش دهید (مثلاً 0.5).

📊 نظارت و مانیتورینگ (Monitoring)

استفاده از TensorBoard
مشاهده real-time پیشرفت آموزش، نمودار loss ها و متریک‌ها.
# اجرای TensorBoard tensorboard --logdir runs/ --port 6006 # در مرورگر باز کنید # http://localhost:6006
📈 متریک‌های قابل مشاهده:
  • Training Loss / Validation Loss
  • Learning Rate Schedule
  • Mel Spectrogram Samples
  • Audio Waveform Samples
  • GPU Utilization
مانیتورینگ GPU
بررسی استفاده بهینه از GPU در حین آموزش.
# مانیتورینگ real-time GPU watch -n 1 nvidia-smi # یا استفاده از اسکریپت مخصوص python3 monitoring/monitor_gpu_live.py
لاگ‌های آموزش
تمام اطلاعات آموزش در فایل log ذخیره می‌شود.
# مشاهده لاگ‌های زنده tail -f training.log # جستجوی خطاها grep "ERROR" training.log # جستجوی بهترین loss grep "Best" training.log

🔍 رفع مشکلات (Troubleshooting)

مشکل: Loss به NaN تبدیل می‌شود
⚠️ علت: Exploding gradients یا learning rate خیلی بالا
✅ راه‌حل:
  1. Learning rate را کاهش دهید (مثلاً نصف کنید)
  2. Gradient clipping فعال کنید: --gradient-clip-val 0.5
  3. Mixed precision غیرفعال کنید اگر فعال است
  4. Batch size را کاهش دهید
مشکل: Loss خیلی کند پایین می‌آید
⚠️ علت: Learning rate خیلی پایین یا تنظیمات نامناسب
✅ راه‌حل:
  1. Learning rate را افزایش دهید: --lr 0.0005
  2. از optimization level بالاتر استفاده کنید: --optimization-level enhanced
  3. GPU stabilizer فعال کنید: --enable-gpu-stabilizer
  4. Batch size را افزایش دهید
  5. از تنظیمات fast convergence استفاده کنید
مشکل: Out of Memory (OOM)
⚠️ علت: حافظه GPU کافی نیست
✅ راه‌حل:
  1. Batch size را کاهش دهید
  2. Mixed precision فعال کنید: --mixed-precision
  3. Gradient accumulation استفاده کنید: --gradient-accumulation-steps 4
  4. Model size کوچک‌تر انتخاب کنید: --model-size small
مشکل: GPU Utilization پایین است
⚠️ علت: CPU bottleneck یا data loading کند
✅ راه‌حل:
  1. GPU stabilizer فعال کنید: --enable-gpu-stabilizer
  2. تعداد data workers افزایش دهید: --num-workers 8
  3. از precomputed cache استفاده کنید
  4. Batch size را افزایش دهید
مشکل: کیفیت صدای تولیدی پایین است
⚠️ علت: آموزش ناکافی یا Vocoder آموزش ندیده
✅ راه‌حل:
  1. مدل را بیشتر آموزش دهید (حداقل 100 epoch)
  2. اطمینان حاصل کنید Vocoder به درستی آموزش دیده
  3. کیفیت دیتاست را بررسی کنید
  4. از loss weights مناسب استفاده کنید

✨ بهترین روش‌ها (Best Practices)

💾 ذخیره Checkpoint ها
همیشه checkpoint های منظم ذخیره کنید تا در صورت مشکل بتوانید از آن‌ها استفاده کنید.
💡 توصیه:
  • هر 5-10 epoch یک checkpoint ذخیره کنید
  • بهترین checkpoint را نگه دارید (کمترین validation loss)
  • چند checkpoint آخر را برای مقایسه نگه دارید
📊 ارزیابی منظم
به طور منظم کیفیت مدل را روی validation set ارزیابی کنید.
# ارزیابی مدل python3 utilities/evaluate_tts.py \ --checkpoint checkpointsmain/best_model.pt \ --test-data ../dataset/dataset_eval
🎯 استراتژی آموزش تدریجی
شروع با تنظیمات ساده و به تدریج پیچیده‌تر کردن.
مراحل توصیه شده:
  1. مرحله 1: آموزش پایه با تنظیمات default (20-30 epoch)
  2. مرحله 2: فعال‌سازی optimizations (30-50 epoch)
  3. مرحله 3: Fine-tuning با learning rate کمتر (50+ epoch)
  4. مرحله 4: آموزش نهایی تا همگرایی کامل
🔄 Backup و Version Control
همیشه از کد و checkpoint های خود backup بگیرید.
💡 توصیه:
  • از Git برای version control کد استفاده کنید
  • Checkpoint های مهم را در مکان امن ذخیره کنید
  • تنظیمات و پارامترهای موفق را یادداشت کنید
📝 مستندسازی آزمایش‌ها
نتایج و تنظیمات هر آزمایش را ثبت کنید.
# ایجاد فایل log برای هر آزمایش python3 train_main.py \ --train-data ../dataset/dataset_train \ --val-data ../dataset/dataset_eval \ --experiment-name "exp_001_baseline" \ --log-file logs/exp_001.log
